# 文档格式规范

## 适用范围
本规范适用于 `docs/` 目录下所有 Markdown 文件。

## 标题结构
- **主标题**：`# 1. 主题名称`
- **二级标题**：`## 1.1. 子主题`
- **三级标题**：`### 1.1.1. 细分主题`
- **四级标题**：`#### 1.1.1.1. 具体技术`

## 内容组织
每个文档应包含以下板块（根据主题适当调整）：
1. **学习资料**：教程、课程、博客等
2. **开源工具/模型**：GitHub 仓库、框架等
3. **研究机构**：实验室、团队等
4. **核心技术**：按技术方向细分
5. **细分应用**：特定领域应用
6. **相关理解**：概念解析、方法论等

## 格式规范
- **链接**：`[链接文本](URL) 简短描述`
- **列表**：使用 `-` 符号
- **论文**：`[年份.月份] [论文标题](URL) 简要说明`
- **项目**：`- [项目名](URL) 描述`
- **空行**：不同部分之间用空行分隔
- **排序**：论文按年份倒序排列

## 内容要求
- 只收录经过验证的优质资源
- 提供清晰的分类和层次结构
- 保持内容简洁明了
- 定期更新维护

请参考 `docs/4.[模型]文本/PostTraining.md` 和 `docs/4.[模型]文本/CodeLLM.md` 的格式。
